  • First commit in 2005
  • Distributed version control system
  • Hugely popular with open source projects
*Git is a DVCS *As a developer I've witnessed it unstoppable rise in popularity. *Especially in open source, helped no end by GitHub. *But as widespread as it is, it's still widely regarded as impenatrable by many *At it's very core it is based on a simple idea though

Sequence of commits

(containing files)

*All a version control system is is a sequence of snapshots of a folder of files *You can replicate much of the functionality by saving a file with a timestamp after every change *Let's look at one of these snapshots, called a commit 

$ git cat-file -p e83c5163316f89bfbde7d9ab23ca2e25604af290
tree 2b5bfdf7798569e0b59b16eb9602d5fa572d6038
author Linus Torvalds <torvalds@ppc970.osdl.org> 1112911993 -0700
committer Linus Torvalds <torvalds@ppc970.osdl.org> 1112911993 -0700

Initial revision of "git", the information manager from hell
*Immediately you can see how intimidating it looks *But look beyond that and you can see this commit is identified by the 40 character id starting e83c *And the file snapshot is stored in another object identified by the other 40 character id starting 2b5b *The rest is just metadata, including a message describing the change. *This is actually the commit that contained the first ever version of git *And you can see that it's creator Linus Torvalds shares the popular opinion of git 

$ git cat-file -p 8bc9a0c769ac1df7820f2dbf8f7b7d64835e3c68

Add copyright notices.

The tool interface sucks (especially "committing" information, which is just
me doing everything by hand from the command line), but I think this is in
theory actually a viable way of describing the world. So copyright it.
*This is the second commit ever made, you'll spot Linus is getting more specific in his critique *But look a bit harder and see the extra field referencing parent *That hash (again starting e83c) is the identifier of the previous commit *This is what links the commits together into a sequence
*And here we see the frightening limits of my artistic ability *The first commit we saw is on the right (no previous/parent reference) *The second commit is in the middle with a reference to the first. *And the third is just there to show how this pattern continues. *And that's all there is to it!

  • First block mined in 2009
  • Distributed cryptographic currency
  • Hugely popular with purchasers of ilicit substances
*bitcoin is a distributed cryto-currency *it's a bit newer than git, and in fact uses git for hosting it's source code *probably most famous for alternating headlines of people making and losing lots of cash and speculation of if being the next big thing *2,000x smaller than UK, 10,000x smaller than China *Currently still a very niche concept, despite the headlines. *Very similar to git, seemingly impenitrable from the outside but actually based around a simple idea

Sequence of blocks

(containing transactions)

*As strange as it may seem the hardest part of bitcoin isn't making transactions but keeping a ledger of them *Instead of storing each transaction as a separate item in the sequence, bitcoin groups them in 10min chunks called blocks *The ledger is therefore a sequence of these blocks

So???

*So matryoshka dolls have the same pattern as git and bitcoin but why does that matter?
*Because you can hold onto matryoshka dolls! *One thing I didn't mention about git, bitcoin and sequences in general is that order matters *I know from personal experience git history graphs look complicated enough without them re-arranging themselves *And in bitcoin you could easily end up the equivalent of overdrawn *The hidden trick all 3 systems demonstrate is a secure representation of a sequence. *If I hold the outside doll, I know that no one can manipulate any of the dolls inside it. *By extension if I have the latest bitcoin block or git commit, * I know that no one can change any of the previous blocks or commits without me noticing.
